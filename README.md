# BloodLifeSaver – Project Proposal (MVP)

## Project Overview

**BloodLifeSaver** is a web-based platform designed to connect **blood donors**, **hospitals**, and **administrators**. The MVP will simplify blood donation management by allowing donors to register, hospitals to request specific blood types, and admins to oversee and approve requests. The project demonstrates full-stack integration using **Next.js**, **Node.js**, and **Supabase**, with automated build and testing through **GitLab CI/CD**.

---

## Objectives

- Develop a functional prototype for blood donation management.
- Integrate donor registration, hospital requests, and admin management modules.
- Implement automated build, test, and deploy pipelines using GitLab CI/CD.
- Ensure responsive UI and smooth backend communication.

---

## System Components (MVP Scope)

| Module                    | Description                                                         |
| ------------------------- | ------------------------------------------------------------------- |
| **Donor Module**          | Donors register with name, age, blood type, and last donation date. |
| **Hospital Module**       | Hospitals request specific blood types and view their requests.     |
| **Admin Dashboard**       | Admin manages donors and blood requests and updates statuses.       |
| **Integration & Testing** | Connect frontend, backend, and database; test via Jest and Postman. |

---

## Tech Stack

| Layer               | Technology                                       |
| ------------------- | ------------------------------------------------ |
| **Frontend**        | Next.js + Tailwind CSS (Deployed on **Netlify**) |
| **Backend**         | Node.js + Express (Deployed on **Render**)       |
| **Database**        | Supabase (PostgreSQL)                            |
| **Version Control** | Git (via GitLab)                                 |
| **CI/CD**           | GitLab CI Pipelines                              |
